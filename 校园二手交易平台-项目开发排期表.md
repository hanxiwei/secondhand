# 校园二手交易平台项目开发排期表

## 1. 文档说明

### 1.1 文档目的
本文件用于规划校园二手交易平台 V1.0 的实施节奏、阶段目标、任务安排和交付物，帮助项目按阶段推进并控制开发范围。

### 1.2 排期假设
以下排期以 `8 周 MVP 版本` 为基准，默认前后端分离开发，团队最少配置如下：

1. 产品/项目负责人 1 人。
2. UI 设计 1 人。
3. 前端开发 1 到 2 人。
4. 后端开发 1 到 2 人。
5. 测试 1 人。

若团队规模更小，可将总周期调整为 10 到 12 周。

## 2. 总体排期概览

| 周次 | 阶段 | 目标 |
| --- | --- | --- |
| 第 1 周 | 需求与设计 | 明确范围、产出产品与技术设计文档 |
| 第 2 周 | 项目初始化 | 搭建前后端基础框架和开发规范 |
| 第 3 周 | 用户与认证 | 完成登录注册、鉴权、个人中心基础能力 |
| 第 4 周 | 商品核心功能 | 完成商品发布、编辑、列表、详情、搜索筛选 |
| 第 5 周 | 收藏消息交易 | 完成收藏、会话消息、订单状态流转 |
| 第 6 周 | 管理后台 | 完成商品审核、举报处理、分类和公告管理 |
| 第 7 周 | 联调测试 | 完成联调、问题修复、数据验证和体验优化 |
| 第 8 周 | 发布上线 | 完成部署、验收、培训和上线准备 |

## 3. 分阶段任务安排

### 第 1 周：需求与设计

#### 阶段目标
明确项目边界，完成产品、技术和数据层面的设计准备。

#### 核心任务
1. 确认 MVP 范围和非范围内容。
2. 完成规范目标文档确认。
3. 完成 PRD 文档。
4. 完成数据库设计文档。
5. 完成接口设计文档。
6. 完成页面清单和原型草图。
7. 确定技术栈和部署方式。

#### 交付物
1. `校园二手交易平台-规范目标文档.md`
2. `校园二手交易平台-PRD产品需求文档.md`
3. `校园二手交易平台-数据库表结构设计文档.md`
4. `校园二手交易平台-接口设计文档.md`
5. 页面原型图。
6. ER 图。

### 第 2 周：项目初始化

#### 阶段目标
完成前后端基础工程搭建，建立统一开发规范。

#### 核心任务
1. 初始化前端项目。
2. 初始化管理端项目。
3. 初始化后端项目。
4. 配置数据库连接、Redis、文件上传方案。
5. 搭建基础目录结构和分层架构。
6. 配置统一返回结构、异常处理和日志方案。
7. 配置 Swagger/OpenAPI。
8. 建立 Git 分支策略与开发规范。

#### 交付物
1. 前端基础工程。
2. 管理端基础工程。
3. 后端基础工程。
4. 基础环境配置说明。
5. README 和启动说明。

### 第 3 周：用户与认证模块

#### 阶段目标
打通用户注册、登录、鉴权和个人资料能力。

#### 核心任务
1. 完成用户表和认证相关表建表。
2. 开发注册接口、登录接口、退出接口、重置密码接口。
3. 开发 Token 鉴权和权限中间件。
4. 开发获取当前用户信息和修改资料接口。
5. 完成前端登录页、注册页、个人中心页。
6. 完成头像上传能力。
7. 编写认证和用户模块基础测试。

#### 交付物
1. 认证模块接口。
2. 用户中心基础页面。
3. 登录态管理方案。

### 第 4 周：商品核心模块

#### 阶段目标
完成商品发布、编辑、展示、浏览和搜索能力。

#### 核心任务
1. 完成分类表、商品表、商品图片表建表。
2. 开发分类查询接口。
3. 开发商品发布、编辑、删除、上下架接口。
4. 开发商品列表、详情、搜索、筛选、排序接口。
5. 开发商品图片上传接口。
6. 完成首页、商品列表页、商品详情页、发布商品页、编辑商品页。
7. 实现浏览量和基础统计更新逻辑。

#### 交付物
1. 商品模块接口。
2. 商品端页面。
3. 分类数据初始化脚本。

### 第 5 周：收藏、消息、交易模块

#### 阶段目标
打通用户互动和交易状态闭环。

#### 核心任务
1. 完成收藏表、会话表、消息表、交易表建表。
2. 开发收藏和取消收藏接口。
3. 开发会话列表、消息发送、消息已读接口。
4. 开发发起购买、确认交易、取消交易、完成交易接口。
5. 完成我的收藏页、消息中心页、我的订单页。
6. 处理商品状态与订单状态联动。
7. 完成关键流程联调。

#### 交付物
1. 收藏模块接口。
2. 消息模块接口。
3. 交易模块接口。
4. 关键闭环演示版本。

### 第 6 周：管理后台与治理能力

#### 阶段目标
完成后台核心运营和内容治理功能。

#### 核心任务
1. 完成管理员表、举报表、公告表建表。
2. 开发后台登录接口。
3. 开发用户列表、冻结解冻接口。
4. 开发商品审核、商品下架接口。
5. 开发分类管理接口。
6. 开发举报处理接口。
7. 开发公告管理接口。
8. 完成管理端页面开发。

#### 交付物
1. 管理端可登录版本。
2. 商品审核和举报处理功能。
3. 分类与公告管理功能。

### 第 7 周：联调测试与优化

#### 阶段目标
完成系统联调、缺陷修复和基础体验优化。

#### 核心任务
1. 前后端全模块联调。
2. 完成核心流程测试。
3. 修复阻断级和高优先级缺陷。
4. 优化商品列表性能和搜索体验。
5. 完善异常提示和空状态展示。
6. 补充测试数据和基础运维脚本。
7. 输出测试报告。

#### 交付物
1. 测试报告。
2. 缺陷清单和修复记录。
3. 候选上线版本。

### 第 8 周：部署上线

#### 阶段目标
完成部署、验收和上线准备。

#### 核心任务
1. 配置测试环境和正式环境。
2. 完成数据库初始化和数据备份方案。
3. 编写部署文档和启动脚本。
4. 完成上线前验收。
5. 完成使用说明和后台操作说明。
6. 上线并观察首轮运行情况。

#### 交付物
1. 部署文档。
2. 环境配置说明。
3. 上线版本清单。
4. 运维交接资料。

## 4. 模块优先级安排

### P0 必做
1. 注册登录。
2. 用户中心。
3. 商品发布与管理。
4. 商品浏览与搜索。
5. 收藏。
6. 消息沟通。
7. 交易状态。
8. 后台商品审核。
9. 举报处理。

### P1 建议做
1. 公告管理。
2. 数据统计看板。
3. 用户冻结与解冻。

### P2 可延后
1. 评论。
2. 点赞。
3. 推荐算法。
4. 小程序端支持。

## 5. 人员分工建议

2. 跟踪排期和风险。
3. 组织验收和评审。

### UI 设计
1. 输出用户端和管理端页面设计稿。
2. 输出组件样式和交互规范。

### 前端开发
1. 负责用户端和管理端页面实现。
2. 负责接口联调和状态管理。
3. 负责组件封装和页面交互优化。

### 后端开发
1. 负责数据库设计和接口开发。
2. 负责鉴权、权限、上传、日志和业务逻辑实现。
3. 负责部署配置和运行环境支持。

1. 编写测试用例。
2. 执行功能测试和回归测试。
3. 跟踪缺陷关闭情况。

## 6. 里程碑检查点

### M1：需求冻结
时间：第 1 周末

检查项：
1. 四份核心文档输出完成。
2. MVP 范围确认。
3. 技术栈确认。

### M2：基础框架完成
时间：第 2 周末

检查项：
1. 前后端项目可启动。
2. 数据库可连接。
3. 文档与基础工程一致。

### M3：用户与商品主链路完成
时间：第 4 周末

检查项：
1. 用户可登录并发布商品。
2. 商品可被浏览和搜索。
3. 商品详情可正常展示。

### M4：交易闭环完成
时间：第 5 周末

检查项：
1. 用户可收藏商品。
2. 买卖双方可沟通。
3. 交易状态可流转。

### M5：后台治理完成
时间：第 6 周末

检查项：
1. 后台可审核商品。
2. 举报可处理。
3. 分类和公告可管理。

### M6：上线验收完成
时间：第 8 周末

检查项：
1. 核心流程可稳定运行。
2. 无阻断级缺陷。
3. 部署和回滚方案明确。

## 7. 风险与缓冲

### 7.1 需求膨胀风险
处理方式：
严格按照 P0、P1、P2 管理需求，避免首期范围失控。

### 7.2 联调延期风险
处理方式：
前后端接口先行，优先定义 Mock 数据和联调用例。

### 7.3 技术复杂度风险
处理方式：
第一阶段不引入支付、推荐、实时 IM 等复杂能力。

### 7.4 人员不足风险
处理方式：
优先保障用户认证、商品和交易闭环，后台统计类功能可后置。

## 8. 推荐项目推进方式
1. 每周固定一次需求和进度评审。
2. 每个模块开发前先确认接口。
3. 每周输出阶段版本。
4. 测试尽量从第 3 周开始提前介入。

6. 关键接口文档可访问。
