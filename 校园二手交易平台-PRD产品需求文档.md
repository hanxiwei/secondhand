# 校园二手交易平台 PRD 产品需求文档

## 1. 文档说明

### 1.1 文档目的
本文件用于明确校园二手交易平台第一阶段产品需求，统一产品、设计、前端、后端和测试的认知，作为页面设计、接口设计、数据库设计和项目排期的依据。

### 1.2 产品版本
V1.0 MVP 版本

### 1.3 适用范围
本期范围聚焦校园内二手商品交易闭环，包含用户注册登录、商品发布与浏览、站内沟通、交易状态管理、后台审核与基础运营能力。

## 2. 产品背景

### 2.1 背景说明
校园内二手交易场景普遍存在，但目前很多交易通过群聊、朋友圈、论坛或临时沟通工具完成，存在信息分散、信任度不足、查询困难和管理缺失的问题。需要建设一个面向校园用户的统一平台，帮助用户高效完成闲置物品的发布、查找和沟通。

### 2.2 产品愿景
打造一个安全、清晰、便捷的校园二手交易平台，让用户能够快速发布商品、准确找到目标商品，并在校内可信环境下完成交易。

### 2.3 产品目标
1. 建立校园内可持续运营的二手交易基础平台。
2. 形成完整的二手商品展示和交易闭环。
3. 为后续接入评价体系、推荐系统和移动端扩展提供基础。

## 3. 用户与角色

### 3.1 目标用户
1. 有闲置物品出售需求的在校学生。
2. 有低价购买需求的在校学生。
3. 负责平台审核、运营和内容管理的管理员。

### 3.2 角色划分

#### 普通用户
主要完成注册、登录、浏览商品、发布商品、收藏商品、发消息沟通、查看交易记录和举报违规内容。

#### 管理员
主要完成用户管理、商品审核、举报处理、分类管理、公告配置和基础数据统计。

## 4. 业务范围

### 4.1 本期必须实现
1. 用户注册、登录、退出登录。
2. 校园身份基础校验。
3. 个人资料维护。
4. 商品发布、编辑、删除、上下架。
5. 商品浏览、搜索、筛选、排序。
6. 商品收藏与取消收藏。
7. 买卖双方站内消息沟通。
8. 交易状态流转。
9. 举报与后台审核处理。
10. 后台用户、商品、分类、公告、举报管理。

### 4.2 本期不实现
1. 在线支付。
2. 担保交易。
3. 物流跟踪。
4. 智能推荐。
5. 评论与社交裂变体系。
6. 多校区联盟交易平台。

## 5. 关键业务流程

### 5.1 用户购买流程
1. 用户注册并登录平台。
2. 用户浏览首页或搜索目标商品。
3. 用户查看商品详情。
4. 用户通过站内消息联系卖家。
5. 用户发起购买意向。
6. 卖家确认后，系统记录交易中状态。
7. 线下面交完成后，卖家将商品标记为已售出。
8. 系统记录交易完成状态。

### 5.2 用户发布流程
1. 用户登录平台。
2. 进入发布商品页面。
3. 填写标题、分类、价格、成色、描述、图片等信息。
4. 提交商品。
5. 商品进入已发布或待审核状态。
6. 审核通过后对外展示。

### 5.3 举报处理流程
1. 用户发现违规商品或违规用户。
2. 提交举报原因和说明。
3. 系统记录举报单。
4. 管理员在后台处理举报。
5. 对应商品或用户被下架、警告或冻结。

## 6. 功能需求明细

### 6.1 认证模块

#### 6.1.1 功能目标
为平台提供安全稳定的注册、登录和身份识别能力。

#### 6.1.2 功能点
1. 用户注册。
2. 用户登录。
3. 退出登录。
4. 重置密码。
5. 校园身份校验。
6. Token 刷新。

#### 6.1.3 业务规则
1. 用户名、手机号、邮箱至少一种可作为登录标识。
2. 手机号和邮箱应保证唯一。
3. 密码必须加密存储。
4. 校园身份未完成认证的用户可限制部分功能。

#### 6.1.4 验收标准
1. 用户可成功完成注册和登录。
2. 登录后可访问需要鉴权的接口。
3. 非法账号或密码返回明确错误信息。

### 6.2 用户中心模块

#### 6.2.1 功能目标
为用户提供个人资料维护和个人资产查看入口。

#### 6.2.2 功能点
1. 查看个人资料。
2. 编辑头像、昵称、年级、学院、联系方式。
3. 查看我的发布。
4. 查看我的收藏。
5. 查看我的订单。
6. 查看系统通知和会话消息。

#### 6.2.3 业务规则
1. 用户只能编辑自己的资料。
2. 联系方式应支持脱敏显示。
3. 头像支持图片上传，需限制格式和大小。

#### 6.2.4 验收标准
1. 用户资料修改后可立即生效。
2. 我的发布、收藏、订单数据展示准确。

### 6.3 商品管理模块

#### 6.3.1 功能目标
支持卖家对商品进行完整的生命周期管理。

#### 6.3.2 功能点
1. 发布商品。
2. 编辑商品。
3. 删除商品。
4. 上架商品。
5. 下架商品。
6. 查看商品详情。

#### 6.3.3 商品核心字段
1. 标题。
2. 描述。
3. 分类。
4. 售价。
5. 原价。
6. 成色。
7. 图片列表。
8. 所在校区。
9. 交易方式。
10. 联系方式。
11. 商品状态。

#### 6.3.4 业务规则
1. 标题不能为空，且长度需要限制。
2. 商品至少上传一张图片。
3. 价格必须大于零。
4. 已成交商品不可再次上架。
5. 用户只能编辑和删除自己的商品。

#### 6.3.5 验收标准
1. 商品发布成功后可在商品列表展示。
2. 商品编辑后详情页数据同步更新。
3. 商品上下架状态在前台可正确反映。

### 6.4 商品浏览与搜索模块

#### 6.4.1 功能目标
帮助用户快速发现符合条件的商品。

#### 6.4.2 功能点
1. 首页推荐商品。
2. 分类浏览。
3. 商品列表分页展示。
4. 关键词搜索。
5. 多条件筛选。
6. 排序功能。

#### 6.4.3 筛选条件
1. 分类。
2. 价格区间。
3. 发布时间。
4. 校区。
5. 成色。
6. 交易方式。

#### 6.4.4 排序规则
1. 默认按发布时间倒序。
2. 支持按价格升序、价格降序、热度排序。

#### 6.4.5 验收标准
1. 关键词搜索结果准确。
2. 筛选与排序可叠加使用。
3. 列表分页和总数统计准确。

### 6.5 收藏模块

#### 6.5.1 功能目标
帮助用户保存感兴趣商品，提升后续转化。

#### 6.5.2 功能点
1. 收藏商品。
2. 取消收藏。
3. 查看我的收藏列表。

#### 6.5.3 业务规则
1. 同一用户不能重复收藏同一商品。
2. 下架商品在收藏列表中保留状态提示。

#### 6.5.4 验收标准
1. 收藏状态切换正确。
2. 收藏列表与详情页数据一致。

### 6.6 消息沟通模块

#### 6.6.1 功能目标
支持买卖双方围绕商品进行站内沟通。

#### 6.6.2 功能点
1. 发起会话。
2. 发送文本消息。
3. 查看会话列表。
4. 查看未读消息数。
5. 系统通知。

#### 6.6.3 业务规则
1. 消息必须与商品或系统通知关联。
2. 用户只能查看与自己相关的会话。
3. 第一阶段采用站内信或轮询方案即可。

#### 6.6.4 验收标准
1. 用户可成功发送和接收消息。
2. 未读状态和已读状态切换正确。

### 6.7 交易管理模块

#### 6.7.1 功能目标
支持平台记录交易意向与成交结果。

#### 6.7.2 功能点
1. 发起购买意向。
2. 卖家确认交易。
3. 卖家取消交易。
4. 标记商品已售出。
5. 买家查看交易状态。
6. 查看交易记录。

#### 6.7.3 状态定义
1. 待沟通。
2. 交易中。
3. 已成交。
4. 已取消。
5. 已关闭。

#### 6.7.4 业务规则
1. 一个商品同一时间只允许一个有效成交流程。
2. 已售出商品不可再次发起新交易。
3. 交易状态变更必须记录时间和操作者。

#### 6.7.5 验收标准
1. 用户可查看自己相关的交易记录。
2. 交易状态变更准确且可追溯。

### 6.8 举报模块

#### 6.8.1 功能目标
建立基础内容治理和违规处理能力。

#### 6.8.2 功能点
1. 举报商品。
2. 举报用户。
3. 填写举报原因。
4. 查看举报提交结果。

#### 6.8.3 业务规则
1. 举报内容需包含举报对象和原因。
2. 同一用户短时间内频繁举报应做限制。
3. 举报单进入后台待处理列表。

#### 6.8.4 验收标准
1. 举报可正常提交并进入后台。
2. 管理员可查看和处理举报记录。

### 6.9 管理后台模块

#### 6.9.1 用户管理
1. 查看用户列表。
2. 查看用户详情。
3. 冻结和解冻用户。
4. 查看用户违规记录。

#### 6.9.2 商品管理
1. 查看商品列表。
2. 审核商品内容。
3. 下架违规商品。
4. 查看商品举报记录。

#### 6.9.3 分类管理
1. 新增分类。
2. 编辑分类。
3. 启用和禁用分类。
4. 分类排序配置。

#### 6.9.4 公告管理
1. 发布公告。
2. 编辑公告。
3. 上下线公告。

#### 6.9.5 数据统计
1. 用户增长统计。
2. 商品发布统计。
3. 成交统计。
4. 举报统计。

## 8. 关键页面原型说明

### 8.1 首页
展示搜索框、分类导航、推荐商品、最新商品和公告信息。

### 8.2 商品详情页
展示商品图片、标题、价格、成色、描述、卖家信息、收藏按钮、联系卖家按钮和举报按钮。

### 8.3 发布商品页
支持标题、价格、分类、成色、校区、交易方式、图片、描述等内容录入。

### 8.4 消息中心页
展示会话列表、未读数和当前聊天内容。

### 8.5 管理后台
展示数据概览、待审核列表和举报待处理项。

## 9. 非功能需求

### 9.1 性能要求
1. 首页列表接口响应目标不高于 1 秒。
2. 常规接口目标响应时间控制在 500 毫秒到 1500 毫秒。
3. 支持平台初期 1000 名以上注册用户。

### 9.2 安全要求
1. 密码加密存储。
2. 接口统一鉴权。
3. 文件上传校验类型和大小。
4. 防止 SQL 注入、XSS 和越权访问。
5. 敏感操作记录审计日志。

### 9.3 可用性要求
1. 核心流程操作路径短。
2. 异常状态有明确提示。
3. 商品状态对用户清晰可见。

## 10. MVP 验收指标
1. 用户可完成注册、登录、发布商品、浏览商品、发送消息和标记成交。
2. 平台支持管理员审核商品和处理举报。
3. 核心流程无阻断级缺陷。
4. 接口、数据库和页面实现与设计文档一致。
