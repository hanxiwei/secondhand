# 校园二手交易平台接口设计文档

## 1. 文档说明

### 1.1 文档目的

本文件用于定义校园二手交易平台 V1.0 的前后端接口规范，作为前后端联调、接口开发、测试验收和接口文档生成的依据。

### 1.2 接口风格

统一采用 `RESTful API` 风格，数据格式使用 `JSON`，字符集统一使用 `UTF-8`。

### 1.3 基础约定

1. 用户端接口统一前缀为 `/api`。
2. 管理端接口统一前缀为 `/api/admin`。
3. 需要登录的接口统一通过 `Authorization: Bearer <token>` 传递令牌。
4. 上传接口采用 `multipart/form-data`。

## 2. 通用规范

### 2.1 通用返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.2 分页返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "page": 1,
    "pageSize": 10,
    "total": 0
  }
}
```

### 2.3 通用错误码建议

| 错误码 | 含义             |
| ------ | ---------------- |
| 0      | 成功             |
| 40000  | 请求参数错误     |
| 40100  | 未登录或令牌失效 |
| 40300  | 无权限访问       |
| 40400  | 资源不存在       |
| 40900  | 数据冲突         |
| 42900  | 请求过于频繁     |
| 50000  | 服务器内部错误   |

## 3. 鉴权说明

### 3.1 用户端鉴权

以下接口默认需要登录：

1. 个人中心相关接口。
2. 商品发布与管理接口。
3. 收藏接口。
4. 消息接口。
5. 订单接口。
6. 举报接口。

### 3.2 管理端鉴权

管理端全部接口默认需要管理员登录。敏感操作建议增加角色校验。

## 7. 商品模块接口

### 7.1 分页查询商品列表

- 方法：`GET`
- 路径：`/api/products`
- 鉴权：否

查询参数：

| 参数名         | 类型    | 必填 | 说明                                          |
| -------------- | ------- | ---- | --------------------------------------------- |
| page           | int     | 否   | 页码，默认 1                                  |
| pageSize       | int     | 否   | 每页条数，默认 10                             |
| keyword        | string  | 否   | 搜索关键词                                    |
| categoryId     | long    | 否   | 分类 ID                                       |
| minPrice       | decimal | 否   | 最低价格                                      |
| maxPrice       | decimal | 否   | 最高价格                                      |
| campus         | string  | 否   | 校区                                          |
| conditionLevel | int     | 否   | 成色                                          |
| tradeMethod    | int     | 否   | 交易方式                                      |
| sortBy         | string  | 否   | 排序字段，如 latest、priceAsc、priceDesc、hot |

返回列表项建议字段：

1. `id`
2. `title`
3. `price`
4. `originalPrice`
5. `conditionLevel`
6. `campus`
7. `coverImage`
8. `favoriteCount`
9. `viewCount`
10. `publishedAt`
11. `seller`

## 20. 接口开发顺序建议

1. 认证模块。
2. 用户中心模块。
3. 分类模块。
4. 商品模块。
5. 收藏模块。
6. 消息模块。
7. 交易模块。
8. 举报模块。
9. 管理端商品审核与举报处理。
10. 公告和数据统计模块。

## 21. 接口联调注意事项

1. 所有列表接口统一分页字段。
2. 所有状态字段统一返回枚举值，前端通过字典映射展示。
3. 金额字段统一保留两位小数。
4. 上传接口应返回可直接访问的文件 URL。
5. 接口文档最终建议同步到 Swagger/OpenAPI。
