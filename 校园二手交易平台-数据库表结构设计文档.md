# 校园二手交易平台数据库表结构设计文档

## 1. 文档说明

### 1.1 文档目的
本文件用于定义校园二手交易平台 V1.0 的核心数据库结构，作为后端建模、建表、接口开发和测试数据准备的依据。

### 1.2 数据库选型
建议使用 `MySQL 8.x`，字符集使用 `utf8mb4`，排序规则建议使用 `utf8mb4_general_ci` 或项目统一规则。

### 1.3 命名规范
1. 表名使用小写下划线命名。
2. 主键统一使用 `id`。
3. 时间字段统一使用 `created_at`、`updated_at`。
4. 状态字段统一使用 `status`。
5. 逻辑删除字段统一使用 `is_deleted`。

## 2. 表清单
本期建议核心表如下：

1. `user` 用户表。
2. `user_auth` 用户认证扩展表。
3. `category` 商品分类表。
4. `product` 商品表。
5. `product_image` 商品图片表。
6. `favorite` 收藏表。
7. `message_session` 会话表。
8. `message` 消息表。
9. `trade_order` 交易订单表。
10. `report` 举报表。
11. `notice` 公告表。
12. `admin_user` 管理员表。
13. `operation_log` 操作日志表。

## 3. 设计约定

### 3.1 主键方案
可使用以下两种方案之一：

1. `bigint` 自增主键。
2. 雪花 ID 或其他分布式 ID。

本文档以下示例以 `bigint` 为主。

### 3.2 逻辑删除
业务数据建议保留逻辑删除字段：

1. `0` 表示未删除。
2. `1` 表示已删除。

### 3.3 状态字段
所有状态字段尽量采用数值枚举，前后端通过枚举映射展示文案，避免数据库直接存储中文。

| student_no | varchar(50) | 否 | null | 学号或校验信息 |

5. 普通索引：`idx_user_created_at(created_at)`。

## 5. 表关系说明
1. `user` 与 `product` 是一对多关系。
2. `product` 与 `product_image` 是一对多关系。
3. `user` 与 `favorite` 是一对多关系。
4. `product` 与 `favorite` 是一对多关系。
5. `user` 与 `message_session` 通过买家和卖家字段建立关系。
6. `message_session` 与 `message` 是一对多关系。
7. `product` 与 `trade_order` 是一对多关系。
8. `user` 与 `trade_order` 分别通过买家和卖家字段建立关系。
9. `user` 与 `report` 通过举报人字段建立关系。

## 7. 数据库约束建议
1. 关键唯一字段必须建立唯一索引。
2. 核心列表查询字段必须建立组合索引。
3. 金额字段统一使用 `decimal(10,2)`。
4. 不建议在高并发表中使用过多外键约束，可通过应用层保证完整性。
5. 高频统计字段如浏览量、收藏量可采用异步更新或缓存辅助。
